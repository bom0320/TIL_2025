# mask-image

### mask-image란?

`mask` 아이템을 부분적으로 표시하거나 완전히 표시하지 않을 수도 있는 기능
즉, `mask`는 웹페이지의 요소를 특정 부분만 표시하거나 가리는 데 사용된다. 이미지나 요소의 일부분만 보이게 할 수도 있고, 투명한 부분을 만들어 다른 요소를 보이게 할 수도 있다.

## mask-image 가 하는 일

`mask-image` 는 요소 위에 "투명도 마스크 레이어"를 한 겹 더 씌우는 것이다.
즉,

- 평소에는 : 요소가 이렇게 보임
  - [내가 그린 header 박스]
- `mask-image`를 쓰면 - 그 위에 흑백(또는 알파) 이미지를 올려놓고 - 마스크가 진한 곳 = 보이는 부분, - 마스크가 투명한곳 = 사라지는 부분이 되는 것이다.
  여기서 중요한 포인트는

> 이 gradient의 색(color) 자체는 거의 중요하지 않다.
> -> 즉, 알파 값(투명도) 만 중요하다!!!!!!!

그래서

- `rgb(0, 0, 0)` => 알파가 1(완전 불투명) -> 완전 보임
- `rgba(0, 0, 0, 0)` => 알파가 0 -> 완전 안보임
  즉, 검정이냐 흰색이냐가 중요한 게 아니라 "알파가 1이냐 0이냐"가 핵심이라고 이해하면 된다.

# 예제

```css
mask-image: linear-gradient(
  to right,
  rgba(0, 0, 0, 0) 0%,
  rgb(0, 0, 0) 12.5%,
  rgb(0, 0, 0) 87.5%,
  rgba(0, 0, 0, 0) 100%
);
```

```css
linear-gradient(to bottom, ...)
```

### Gradient(그라디언트)란

CSS 그라디언트란 **둘 이상의 색상을 부드럽게 전환하여 시각적 혀과를 주는 기술**이다.
`linear-gradient()`(선형), `radial-gradient()`(방사형), `conic-gradient()` (원뿔형) 등 다양한 함수를 사용하며, 이를 통해 웹페이지의 배경이나 글자 등에 다채로운 색상 표현을 적용할 수 있다.

**🤔주요 특징 및 종류**

- **점진적인 색상 변화 :** 한 색상에서 다른 색상으로 부드럽게 이어지는 중간 단계를 만들어낸다.
- **다양한 그라데이션 타입 :**

  - 선형 그라데이션(linear-gradient()) : 색상이 직선 방향(수직, 수평, 대각선 등)으로 점진적으로 변한다.
  - 방사형 그라데이션(radial-gradient()) : 색상이 원형으로 중ㅅ미점에서부터 퍼져나간다.
  - 원뿔형 그라데이션(conic-gradient()) : 색상이 중심점을 기준으로 원을 그리며 회전하는 형태로 변한다.

- **CSS3 기술 :** 이전에는 그래픽 프로그램으로만 가능했던 효과를 CSS 코드로 구현할 수 있게 해준다.

## 2. 각 스탑(stop)이 의미하는 범위

이제 이 부분을 구간별로 쪼개 보자

```css
rgb(0 0 0) 0%,
rgb(0, 0, 0) 12.5%,
rgb(0, 0, 0) 57.5%,
rgba(0, 0, 0, 0) 100%
```

#### ▶ 0% ~ 12.5% 구간

- 시작 : `rgb(0 0 0) 0%`
- 끝: `rgb(0 0 0) 12.5%`
  둘 다 같은 색 + 같은 알파(1) -> 즉, 이 구간은 쭉 "완전 보이는 상태"를 유지

> 즉, header의 위쪽 12.5% 영역은 "마스크 100% 불투명 = 전부 보임"

#### ▶ 12.5% ~ 57.5% 구간

- 시작 : `rgb(0, 0, 0) 12.5%`
- 끝 : `rgb(0, 0, 0) 57.5%`
  여기도 마찬가지로 둘 다 같은 값이다 -> header의 12.5% ~ 57.5% 높이까지는 계속 완전 보임 상태

즉, 여기가지 정리하면

> 0% ~ 57.5% : header는 그냥 온전히 보인다.
> (마스크가 100% 불투명 상태)

#### ▶ 57.5% ~ 100% 구간 (여기가 핵심)

- 시작: `rgb(0, 0, 0) 57.5%` -> 불투명
- 끝: `rgba(0, 0, 0, 0) 100%` -> 완전 투명
  그래서 이 구간에서 일어나는 일

> 57.5% 지점에서는 완전 보이다가,
> 100% 지점으로 갈수록 점점 투명해짐

즉,

- header 높이의 57.5% 지점까지는 100% 보이고,
- 그 아래부터는 "점점 흐려지다가" 완전히 사라지는 꼬리 형태
  그래서 실제로는
- 상단 ~ 중간까지느 딱 떨어지는 헤더
- 아래쪽 끝 부분이 자연스럽게 페이드 아웃되는 느낌

## 3. 그렇다면 왜이렇게 12.5, 57.5% 이런식의 애매한 숫자를 사용할까

사실 이 코드는 대충 이렇게 줄여도 거의 같은 효과가 난다.

```css
mask-image: linear-gradient(
  to bottom,
  rgba(0, 0, 0, 1) 0%,
  rgba(0, 0, 0, 1) 57.5%,
  rgba(0, 0, 0, 0) 100%
);
```

- 0%와 12.5%가 같은 값이라
  0~12.5% 사이에 변화가 없기 때문에
  12.5% 스탑은 사실상 “디자인 미세 조정용 잔재” 느낌일 가능성이 큼.

**추측 가능한 의도:**
처음엔 위에서도 살짝 페이드 주려다가(0%를 투명으로 했다가)
나중에 디자인 바꾸면서 0%도 불투명으로 돌려놓고,
중간 스탑(12.5%)은 그냥 남아 있는 상태일 수 있음.

## 4. 이게 실제로 header에 주는 시각적 효과

1. fixed 헤더가 "딱 잘려있는 박스" 처럼 안 보이고
2. 스크롤 되는 컨텐츠랑 부드럽게 이어지는 느낌을 줌
3. 특히 `backdrop-filter: blur(20px)`랑 같이 쓰면 헤더 아래 경계가 아주 자연스럽게 비쳐보이는 느낌
