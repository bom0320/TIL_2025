# useRef

기존에 난 useRef란 불필요한 렌더링을 최소화 하기 위해 사용한다는 것만 알고 있었다. 하지만 최근에 GSAP에 대해 공부하면서 새롭게 알게 된게 있다. 그에 대해 작성해보려고 한다.

useRef는 크게 두가지 주요 용도로 쓰인다.

## 1. 불필요한 렌더링 최소화 (값 저장 용도)

- 이것이 `useRef` 의 가장 일반적인 용도 중 하나이며, 상태(state)와 다른 점이다.
- useRef에 저장된 값(`ref.current`) 은 값이 변경되어도 컴포넌트가 다시 렌더링되지 않는다.
- 예를 들어 타이머 ID, 이전 상태 값, 또는 렌더링 시점에 영향을 주지 않으면서 컴포넌트 생애 주기 동안 유지되어야 하는 변수 등을 저장할 때 사용된다.

## 2. DOM 요소에 직접 접근 (DOM 참조 용도)

- 이것은 React의 "탈출구(Escape Hatch)" 역할을 한다.
- React는 선언적으로 UI를 관리하고 DOM을 직접 조작하지 않는 것을 목표로 하지만, 때로는 실제 DOM 노드에 접근해야 할 필요가 있다.
- 이 필요성은 주로 다음과 같은 상황에서 발생한다.
  - **외부 라이브러리 연동 :** 위에서 언급된 GSAP, D3.js, Chart.js등 React 프레임워크 바깥에서 DOM을 직접 제어하는 라이브러리를 사용할 때
  - **특정 브라우저 API 사용 :** `focus()`, `scrollIntoView()` , `getBoundingClintRect()` 같은 DOM API 메서드를 호출해야 할 때
  - **캔버스 조작 :** `<canvas>` 요소의 컨텍스트에 접근할때

즉, `useRef` 는 **"렌더링에 영향을 주지 않는 지속적인 참조(Persistent Reference)"** 를 제공하는 도구이며, 그 참조 대상이 일반 JS 변수 일 수도 있고, React가 관리하는 실제 DOM 노드일수도 있다.
